git版本控制工具

2021-08-25 06:31:35 Git版本控制工具

Git安装

傻瓜式安装下一步下一步即可，略

Git的结构

Git初始化本地仓库

1、创建一个文件夹

2、执行命令

初始化成功以后会有一个，git的隐藏文件夹

git init #初始化一个本地库

Git 提交到缓存区和提交到本地库命令

1、提交到暂存区

git add 要提交的文件

2、提交到本地库操作

git commit -m "信息描述"

Git查看提交状态命令

查看提交的状态

git status

Git查看日志操作命令

查看提交日志，方式一

展示提交记录的详细信息

git log

查看提交日志，方式二

提交的信息记录在一行显示

git log --pretty=oneline

查看提交日志，方式三

更加简洁的显示方式

git log --oneline

查看提交日志，方式四

git reflog

HEAD@{数字}，指针回到当前这个历史版本需要走多少步

Git版本回退命令

回退到指定版本

git reset --hard ed4b7bf

  • 相关参数

    –hard 参数，使用这个参数的时候，本地库的指针移动的同时，重置暂存区和工作区

    –mixed参数，使用这个参数的时候，本地库的指针移动的同时，重置暂存区，工作区不变

    –soft参数，使用这个参数的时候，本地库的指针移动的同时，暂存区和工作区都不变

Git删除操作

删除工作区的文件

这里仅仅是删除工作区里面的内容，但是暂存区和本地库的内容还会存在

rm 文件名

删除的记录依旧会存在，如果想要找回文件，那么请回退到相应的版本上即可

Git文件对比命令

比对这个文件在工作区和暂存区是否相同

git diff 文件名

比对工作区和暂存区所有的文件

git diff

比对暂存区和本地仓库

git diff [历史版本] [文件名]

Git分支操作

  • 查看分支

    git branch -v

  • 创建分支

    git branch branch01

  • 切换分支

    git checkout branch01

  • 分支合并

    首先要切换在master分支上

    此时这两个分支的版本号不相同这个时候去合并会产生分支冲突的问题

    合并

    git merge branch01

    出现冲突：何时会出现冲突问题？在同一个文件的同一个位置修改

    冲突解决：留下想要的内容，然后在提交

Git将本地库内容提交到远程库

在GitHub 或者码云上创建一个自己的本地仓库，创建过程略

为远程库起别名

git remote add origin https://gitee.com/againnyc/git-test-resp.git

推送操作

git push origin master

Git远程库的拉取操作

git fetch origin master

他只是会将远程库的内容 抓取到本地，但是工作区并不更新。

pull操作相当于fetch和merge操作的合并

Git常用命令操作

  1. 设置签名

    git config --global user.name "签名"

  2. 设置邮箱

    git config --global user.email "邮箱名"

  3. 查git版本

    git version

  4. 清屏操作

    git clear

  5. 本地仓库初始化命令

    git init

  6. 提交到缓存区

    git add 文件名

  7. 提交到本地库

    git commit -m "提交的信息描述"

  8. 查看提交状态的命令

    git status

  9. 查看日志操作

    git log 
git log --pretty=oneline
git log --oneline
git reflog

  10. 版本回退命令

    git reset --hard 要回退的版本索引号
#参数 --hard  --mixed --soft

  11. 删除工作区的命令

    rm 文件名

  12. 文件对比命令

    git diff 文件名
git diff
git diff [历史版本] [文件名]

  13. 查看分支

    git branch -v

  14. 创建分支

    git branch 新的分支名称

  15. 切换分支

    git checkout 要切换的分支名

  16. 合并分支

    git merge 要合并的分支

  17. 为远程库起别名

    git remote add 别名 远程库地址

  18. 推送操作

    git push -u 远程库地址 要推送的分支

  19. 克隆操作

    git clone 远程库地址
